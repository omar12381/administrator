issues

`PUT /forests/{id}` does not re-check intersection/overlap constraints against other forests.
  - `PUT /parcelles/{id}` does not check disjointness against other parcelles (only containment in parent), enabling overlaps after updates.*


improvement
- Add server-side validation for:
    - GeoJSON type must be `Polygon` (and optionally `Feature` wrapping).
    - Ring closure and coordinate array structure.
    - Geometry validity (e.g., Shapely `is_valid`).
  - Enforce invariants in update endpoints:
    - Re-run forest intersection checks on `PUT /forests/{id}`.
    - Re-run parcelle disjointness checks within forest on `PUT /parcelles/{id}`.
  - Improve error handling:
    - Map common exceptions to 4xx (validation) vs 5xx (internal).
    - Avoid returning raw exception strings in production responses.
  - Replace deprecated SQLAlchemy patterns with supported methods (`db.get(Model, id)`).



issues
 - Coordinate units / geodesic accuracy:
    - Area approximation assumes small areas and uses a degrees-to-meters constant without projection transformation; large polygons or varying latitude can produce significant error.
  - Frontend and backend validation mismatch:
    - Frontend overlap logic is heuristic and does not call PostGIS; backend uses spatial predicates (`ST_Contains`, `ST_Disjoint`) that can disagree near boundaries or for invalid geometries.
  - Backend update gap for parcelles:
    - `PUT /parcelles/{id}` validates only containment in forest and recomputes area, but does not enforce non-overlap constraints with other parcelles.
  - Geometry type validation:
    - Backend accepts any dict for `geometry` and converts it; invalid GeoJSON structures may cause Shapely exceptions and lead to 500 errors.



Improvements:
  - Centralize geometry validation in the backend:
    - Validate GeoJSON structure and geometry validity before spatial checks.
  - Ensure update invariants:
    - Add `ST_Disjoint` (or appropriate predicate) checks when updating parcelles to prevent overlap/touching.
  - Improve area accuracy:
    - Consider using PostGIS for area computation in meters (e.g., transform to a projected CRS or use geography-based calculations).
  - Improve performance:
    - Add pagination to list endpoints.
    - Add spatial indexes for geometry columns (and consider selective fields in responses).
  - Add frontend/back-end alignment:
    - Treat frontend checks as UX only; rely on backend as the source of truth for final validation.
